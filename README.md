# SchoolDB — база данных школьной информационной системы

## Описание проекта
SchoolDB — учебный проект реляционной базы данных, разработанный в рамках лабораторной работы по дисциплине «Базы данных».  
База данных моделирует предметную область школьной информационной системы и предназначена для хранения, обработки и анализа данных об учебном процессе, библиотеке и медицинском сопровождении.


---

## Предметная область
База данных отражает основные процессы школьной деятельности:
- учёт учеников и классов;
- хранение информации о преподавателях;
- ведение расписания и учёта оценок;
- медицинские карты и посещения;
- библиотечный фонд и выдача книг;
- учёт родителей (законных представителей);
- аудит изменений данных.

---

## Структура базы данных

### Схема `dict`
Справочные таблицы:
- `Subjects` — учебные предметы;
- `Benefits` — льготы;
- `Rooms` — учебные кабинеты.

### Схема `edu`
Учебный процесс:
- `Teachers` — преподаватели;
- `Classes` — классы;
- `Students` — ученики;
- `Grades` — оценки;
- `Schedule` — расписание занятий.

### Схема `lib`
Библиотека:
- `Books` — книги;
- `BookCopies` — экземпляры книг;
- `Loans` — выдача книг ученикам.

### Схема `med`
Медицинский блок:
- `HealthCards` — медицинские карты учеников;
- `MedVisits` — медицинские посещения.

### Схема `parents`
Родители и законные представители:
- `Parents` — информация о родителях;
- `StudentParents` — связь ученик–родитель (многие ко многим).

### Схема `security`
Безопасность:
- `AuditLog` — журнал аудита изменений данных.

## ER-диаграмма
Для визуализации структуры базы данных была построена ER-диаграмма, отражающая сущности и связи между ними. Диаграмма размещена в репозитории в виде изображения.

---

## Реализованная функциональность
В проекте реализованы:
- представления;
- хранимые процедуры;
- пользовательские функции;
- триггеры для автоматизации и аудита.

---

## Резервное копирование и восстановление
Разработана стратегия резервного копирования:
- полное резервное копирование (Full Backup) — ежедневно;
- дифференциальное резервное копирование (Differential Backup) — ежедневно;
- резервное копирование журнала транзакций (Transaction Log Backup) — каждые 30 минут;
- аварийное резервное копирование Tail-Log Backup — при сбоях.

## Используемые технологии
- СУБД: Microsoft SQL Server;
- Среда разработки: SQL Server Management Studio (SSMS);
- Язык запросов: T-SQL;
- Диаграммы: UML / ER.
